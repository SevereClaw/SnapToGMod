# Snap-to-GMod (модульная версия 3.1)

Слушает микрофон, распознаёт щелчок/хлопок и запускает Garry's Mod через
Steam. Раньше всё это жило в одном файле на 2500+ строк — теперь разбито
на модули по смыслу.

## Структура

| Файл | Отвечает за |
|---|---|
| `config.py` | Пути, константы, `Server`/`SoundEvents`/`AppConfig` (dataclass), загрузка/сохранение |
| `logutil.py` | Настройка `logging` (ротация лога через `RotatingFileHandler`) |
| `system.py` | Права администратора, единственность экземпляра, автозапуск, системные диалоги |
| `servers.py` | CRUD серверов, избранное, поиск, последние использованные, массовая очистка списка, ping + запрос Source Engine |
| `stats.py` | Статистика запусков, экспорт **только в CSV** |
| `sound.py` | Звук по событиям (обнаружен / запуск / ошибка) |
| `audio.py` | Распознавание щелчка и цикл прослушивания микрофона |
| `launcher.py` | Проверка GMod/Steam, запуск игры, вся логика срабатывания |
| `discord_notify.py` | Уведомление в Discord через вебхук |
| `updates.py` | Проверка версии на GitHub, автообновление .exe |
| `hotkeys.py` | Глобальные горячие клавиши |
| `voice_select.py` | Голосовой выбор персонажа по слову-триггеру (офлайн-модель Vosk), несколько персонажей со своими иконками, опционально |
| `tray.py` | Иконка трея, все окна tkinter, сборка меню |
| `main.py` | Точка входа, связывает всё вместе |

Профили настроек убраны (см. ниже), поэтому `profiles.py` из исходного
списка модулей не появился — его обязанности (загрузка/сохранение
единственного конфига) перешли в `config.py`.

## Что убрано

- **Экспорт статистики** — теперь только CSV (были ещё JSON и HTML-отчёт
  с SVG-графиками).
- **Виджет истории рядом с треем** — дублировал журнал и статистику.
- **Профили настроек** — один профиль в `config.json`. При первом запуске
  этой версии активный профиль из старого `profiles.json` (если он был)
  переносится автоматически, остальные профили не сохраняются.
- **Автополучение списка серверов из интернета** (`SERVER_LIST_URL`) —
  вместо этого есть ручное добавление и поиск по имени.

## Что добавлено

- **Поиск сервера по имени** — появляется в меню «Сервер», когда
  серверов больше 8 (`SERVER_SEARCH_THRESHOLD` в `config.py`).
- **Проверка доступности сервера** (ping + A2S_INFO запрос движка Source)
  — вручную из меню и опционально автоматически перед каждым запуском.
- **Избранные сервера** — отмечаются звёздочкой, идут первыми в списке.
- **Очистка списка серверов** — оставляет только избранные и текущий
  активный сервер, остальные удаляет одним пунктом меню (пригодится,
  если список раздулся).
- **«Не запускать, если Steam обновляется»** — эвристика по
  `bootstrap_log.txt` в папке Steam (см. предупреждение ниже).
- **Несколько звуков для разных событий** — «обнаружен щелчок»,
  «успешный запуск», «ошибка запуска» настраиваются по отдельности.
- **Режим «только обнаружение»** — щелчок распознаётся, показывается
  уведомление, игра не запускается.
- **Последние 20–30 использованных серверов** — подменю «Недавние» в
  начале списка серверов.
- **Несколько персонажей в голосовом выборе** — раньше было ровно одно
  слово-триггер и одна захардкоженная иконка (Нагито), из-за чего любое
  слово всегда кликало одного и того же персонажа. Теперь персонажей
  можно завести сколько угодно (менеджер персонажей в трее), у каждого —
  своё слово-триггер и своя иконка; персонажи с одинаковым словом-триггером
  различаются по похожести услышанного текста, а не выбираются по позиции
  на экране наугад.

## Что улучшено

- Файл разбит на модули (см. таблицу выше) вместо одного god object.
- `logging` вместо самодельного логгера — ротация через
  `RotatingFileHandler`.
- Основные константы (`APP_VERSION`, `GITHUB_REPO`, цвета, пресеты
  чувствительности) — в `config.py`; Discord webhook задаётся переменной
  окружения и не публикуется в исходниках.
- Типизация и `dataclass` для `Server`, `SoundEvents`, `AppConfig`,
  `LaunchEntry`/`Stats`, `SourceServerInfo` вместо словарей.
- `pathlib.Path` вместо `os.path` для всех путей.

## Известное ограничение

Официального API для «Steam сейчас обновляется» не существует. Проверка
в `launcher.is_steam_updating()` — эвристика по времени изменения и
содержимому `bootstrap_log.txt` в папке установки Steam; она может
ошибаться и по умолчанию не блокирует запуск, если ничего определить не
удалось. Отключается в меню «Настройки», если будет мешать.

## Запуск

```
pip install -r requirements.txt
python main.py
```

## Discord-уведомления

Webhook нельзя добавлять в публичный репозиторий. Перед запуском задайте его
через переменную окружения Windows PowerShell:

```powershell
$env:SNAPTOGMOD_DISCORD_WEBHOOK_URL = "https://discord.com/api/webhooks/..."
python main.py
```

Для постоянной пользовательской переменной:

```powershell
[Environment]::SetEnvironmentVariable(
    "SNAPTOGMOD_DISCORD_WEBHOOK_URL",
    "https://discord.com/api/webhooks/...",
    "User"
)
```

После изменения постоянной переменной перезапустите программу.

## Голосовой выбор персонажа (опционально)

`voice_select.py` слушает тот же микрофон, что и детектор щелчка (второй
независимый аудиопоток к одному устройству на Windows часто конфликтует),
и по слову-триггеру (офлайн, модель Vosk, без отправки звука куда-либо)
открывает меню паузы и кликает по персонажу через сравнение со
скриншотами-шаблонами.

**Персонажей может быть несколько** — у каждого своё имя (только для вас,
нигде не отображается), своё слово-триггер и своя иконка-шаблон. Два
персонажа могут даже иметь одинаковое слово-триггер (например, общий
талант "Удача" у Нагито Комаэды и Макото Наэги) — при распознавании
выигрывает тот, чьё слово-триггер оказалось ближе всего к услышанному
тексту, а не первый добавленный. Управление — через трей: **Настройки ->
Голосовой выбор персонажа -> Персонажи...** — окно со списком, кнопками
«Добавить персонажа», «Изменить слово-триггер», «Переснять иконку»,
«Удалить». При добавлении первого персонажа окно само проведёт через
захват двух общих для всех персонажей элементов интерфейса (пункт меню
паузы «Выбрать персонажа» и кнопка подтверждения) и иконки самого
персонажа — рамкой прямо на экране игры, без ручного скриншота и
переименования файлов.

Включается через трей: **Настройки -> Голосовой выбор персонажа ->
Включено**. Пока не готовы зависимости — пункт статуса в этом подменю
показывает, чего не хватает — работа остальной программы не затрагивается.

Нужно:
```
pip install -r requirements-voice.txt
```
и модель Vosk (например, `vosk-model-small-ru-0.22`) — скачивается
автоматически кнопкой в меню, либо вручную в `%APPDATA%\SnapToGMod\voice\model`.

### Диагностика и типичная проблема "просто открывается меню паузы"

Подменю трея теперь содержит пункт **«Диагностика и гайд по настройке...»** —
окно, которое проверяет зависимости, модель, файлы-шаблоны (с разрешением
картинки и текущим разрешением экрана), даёт проверить слово-триггер без
микрофона и попробовать найти каждый шаблон на экране прямо сейчас.

Самая частая причина, по которой после слова-триггера в игре только
открывается меню паузы и больше ничего не происходит: **не установлен или
не попал в собранный .exe пакет `opencv-python` (модуль `cv2`)**.
`pyautogui.locateCenterOnScreen(..., confidence=...)` использует `cv2`
внутри себя в момент вызова, а не при импорте — без него сравнение по
confidence всегда возвращает "не найдено", даже если шаблон на экране есть,
и ESC уже нажат. Раньше это было видно только в логе одной строкой.
Диагностика теперь проверяет это отдельно и явно.

Если программа запущена как собранный `.exe`, а не как `python main.py`,
после `pip install -r requirements-voice.txt` **.exe нужно пересобрать**
(`pyinstaller SnapToGMod.spec`) тем же venv, куда ставились зависимости —
иначе `vosk`/`cv2` просто не попадут в сборку (см. комментарий в
`SnapToGMod.spec`).

Если статус в трее долго (дольше ~20 сек) показывает "модель
загружается..." — это само по себе теперь считается зависанием: скорее
всего модель распакована на один уровень глубже, чем нужно (вложенная
папка вроде `model/vosk-model-small-ru-0.22/...` вместо содержимого прямо
в `model/`). Диагностика подсказывает точное имя вложенной папки, которую
надо "поднять".
