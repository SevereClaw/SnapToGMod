# Snap-to-GMod (модульная версия 2.0.0)

Слушает микрофон, распознаёт щелчок/хлопок и запускает Garry's Mod через
Steam. Раньше всё это жило в одном файле на 2500+ строк — теперь разбито
на модули по смыслу.

## Структура

| Файл | Отвечает за |
|---|---|
| `config.py` | Пути, константы, `Server`/`SoundEvents`/`AppConfig` (dataclass), загрузка/сохранение |
| `logutil.py` | Настройка `logging` (ротация лога через `RotatingFileHandler`) |
| `system.py` | Права администратора, единственность экземпляра, автозапуск, системные диалоги |
| `servers.py` | CRUD серверов, избранное, поиск, последние использованные, массовая очистка списка, ping + запрос Source Engine |
| `stats.py` | Статистика запусков, экспорт **только в CSV** |
| `sound.py` | Звук по событиям (обнаружен / запуск / ошибка) |
| `audio.py` | Распознавание щелчка и цикл прослушивания микрофона |
| `launcher.py` | Проверка GMod/Steam, запуск игры, вся логика срабатывания |
| `discord_notify.py` | Уведомление в Discord через вебхук |
| `updates.py` | Проверка версии на GitHub, автообновление .exe |
| `hotkeys.py` | Глобальные горячие клавиши |
| `tray.py` | Иконка трея, все окна tkinter, сборка меню |
| `main.py` | Точка входа, связывает всё вместе |

Профили настроек убраны (см. ниже), поэтому `profiles.py` из исходного
списка модулей не появился — его обязанности (загрузка/сохранение
единственного конфига) перешли в `config.py`.

## Что убрано

- **Экспорт статистики** — теперь только CSV (были ещё JSON и HTML-отчёт
  с SVG-графиками).
- **Виджет истории рядом с треем** — дублировал журнал и статистику.
- **Профили настроек** — один профиль в `config.json`. При первом запуске
  этой версии активный профиль из старого `profiles.json` (если он был)
  переносится автоматически, остальные профили не сохраняются.
- **Автополучение списка серверов из интернета** (`SERVER_LIST_URL`) —
  вместо этого есть ручное добавление и поиск по имени.

## Что добавлено

- **Поиск сервера по имени** — появляется в меню «Сервер», когда
  серверов больше 8 (`SERVER_SEARCH_THRESHOLD` в `config.py`).
- **Проверка доступности сервера** (ping + A2S_INFO запрос движка Source)
  — вручную из меню и опционально автоматически перед каждым запуском.
- **Избранные сервера** — отмечаются звёздочкой, идут первыми в списке.
- **Очистка списка серверов** — оставляет только избранные и текущий
  активный сервер, остальные удаляет одним пунктом меню (пригодится,
  если список раздулся).
- **«Не запускать, если Steam обновляется»** — эвристика по
  `bootstrap_log.txt` в папке Steam (см. предупреждение ниже).
- **Несколько звуков для разных событий** — «обнаружен щелчок»,
  «успешный запуск», «ошибка запуска» настраиваются по отдельности.
- **Режим «только обнаружение»** — щелчок распознаётся, показывается
  уведомление, игра не запускается.
- **Последние 20–30 использованных серверов** — подменю «Недавние» в
  начале списка серверов.

## Что улучшено

- Файл разбит на модули (см. таблицу выше) вместо одного god object.
- `logging` вместо самодельного логгера — ротация через
  `RotatingFileHandler`.
- Все константы (`APP_VERSION`, `GITHUB_REPO`, `DISCORD_WEBHOOK_URL`,
  цвета, пресеты чувствительности) — в `config.py`.
- Типизация и `dataclass` для `Server`, `SoundEvents`, `AppConfig`,
  `LaunchEntry`/`Stats`, `SourceServerInfo` вместо словарей.
- `pathlib.Path` вместо `os.path` для всех путей.

## Известное ограничение

Официального API для «Steam сейчас обновляется» не существует. Проверка
в `launcher.is_steam_updating()` — эвристика по времени изменения и
содержимому `bootstrap_log.txt` в папке установки Steam; она может
ошибаться и по умолчанию не блокирует запуск, если ничего определить не
удалось. Отключается в меню «Настройки», если будет мешать.

## Запуск

```
pip install -r requirements.txt
python main.py
```

## Сборка в .exe

```
pip install pyinstaller
pyinstaller --onefile --noconsole --name SnapToGMod main.py
```

## Скачать готовую сборку

Готовый `SnapToGMod.exe` публикуется в разделе **Releases** после создания
тега версии вида `v2.0.0`.

## Разработка

Рекомендуется Python 3.12:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements-dev.txt
python main.py
```

Сборка через файл спецификации:

```powershell
pyinstaller --clean --noconfirm SnapToGMod.spec
```

## Автоматическая сборка GitHub Actions

Workflow `.github/workflows/build-windows.yml` проверяет синтаксис и собирает
Windows `.exe` при push и pull request. При push тега `v*` исполняемый файл
автоматически прикрепляется к GitHub Release.

Пример выпуска новой версии:

```powershell
git tag v2.0.0
git push origin v2.0.0
```
